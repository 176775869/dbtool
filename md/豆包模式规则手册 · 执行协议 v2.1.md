## 二、状态定义与转移
豆包模式的核心是一个有限状态机。市场在任何时刻都处于且仅处于一种状态。状态转移由基于数据的量化条件触发。

### 2.1 状态速查表

| 状态 | 定义 | 量化触发条件 | 操作指令 |
|------|------|-------------|---------|
| **C** | 启动日 | 连板型：≥2只一字板(或秒板)+≥1只换手板+板块涨>2%。趋势型补充：若无批量涨停，但中军放量涨幅>5%+板块成交创20日新高+板块涨>2%，也视为C。 | 预置锚点。**执行清单为空。** |
| **F** | 发酵期 | C日次日，板块内涨停家数持续增加，中军沿5日线加速上行，先锋封单加大。 | 可加仓至仓2-3。 |
| **G1-n** | 首次主升第n天 | 中军沿5日线上行或滞涨不创新高。n每日递增，**严格跳过非交易日**。 | 持有不加仓。 |
| **D1-n** | 首次分歧第n天 | 盘中出现先锋炸板/中军回踩，**次日确认**。n每日递增。强分歧(D1+)附加：先锋断板无回封+炸板率>40%+中军破5日线。 | D1-1：执行清单为空，筛选活口。 D1-2：尾盘D3低吸(仓2-3)。 D1-3：隔日竞价D2跟随(仓2-3)。 **D1+：减仓预警。** |
| **G2-n** | 二次主升第n天 | 先锋弱转强回封或中军放量修复，**次日确认**。n每日递增。当n≥4时，开始检测BS触发条件。若龙头缩量加速连续2板且板块补涨全面扩散但中军开始滞涨，转入BS。 | 持有不加仓。若转入BS，执行BS对应的操作指令。 |
| **BS** | 博傻期 | G2中n≥4+龙头缩量加速连续2板+板块补涨全面扩散但中军开始滞涨 | 不开新仓，轻仓持有龙头，警惕退潮。检测分离信号(龙头当日涨幅≥5%且板块指数跌幅≥2%)→YG。 |
| **E** | 退潮确认 | 中军破20日线+批量跌停≥3只+评分<3.0连续2日。快速通道(FE)：当日跌停>50家+主线中军跌停+指数跌>5%，三者满足其二，**立即标记E，无需隔日确认**。 | **清仓该板块所有持仓。** |
| **B** | 退潮反弹 | 连跌(上涨家数<1500)≥2天+前主线核心止跌缩量企稳 | 尾盘低吸，隔日套利(仓2)。失败(再创新低)→退回BP。 |
| **BP** | 冰点期 | E后2个交易日窗口期无新C出现+上涨家数连续2日<1500 | 空仓等待。 |
| **Ha** | 混沌期(旧锚点不退潮) | E无法触发，但候选方向持续D级分歧，无法进入G1或G2 | 持有旧锚点观察，不开新仓。候选池多线程跟踪。 |
| **Hb** | 混沌期(无锚点) | E已触发，窗口期已过，但无新C出现，且上涨家数尚未跌破1500 | 空仓等待，不参与反弹。 |
| **YG** | 妖股独舞 | C级环境(指数<20日线+均线空头)+连板≥5板且独立走强(不依赖板块) | 选股≥3板游资独立。买点C(2进3)+D(分歧转一致)。仓2，单票≤30%。断板即走。 |

### 2.2 状态转移规则
1.  **单线递推**：正常周期（C→F→G1→D1→G2→BS→E）是主路径，状态严格按此顺序转移，不允许跳跃。
2.  **隔日确认**：G→D和D→G的切换，**必须**等待下一个交易日数据确认。当日盘中信号不改变当日状态标记。
3.  **交易日历检查**：在每次递推n值之前，**必须**检查当日与上一交易日之间是否有非交易日(周六/周日/法定节假日)。如有，n值不变。不得连续递推。
4.  **强制继承**：从`【历史策略快照】`的全景表中提取主线锚点的最新阶段标记(如G2-3)。今日的阶段标记必须基于此继承值进行递推，不得重新计算。如果今日状态标记未变(如G2→G2)，则n+1；如果状态标记改变(如G2→D1)，则从1重新计数。
5.  **锚点切换缓冲**：挑战者方向（候选池）取代旧锚点，必须同时满足以下三条且持续3天：① 挑战者连续3日评分 ≥ 旧锚点评分；② 挑战者核心中军连续3日走强(收盘价站稳5日线上方)；③ 旧主线核心中军连续3日走弱(收盘价跌破10日线)。若未满足，但候选方向已处于G2阶段，可进入双锚点并行期(最长5天)，并在全景表中明确标注。
6.  **终局分支**：E是正常周期的终点。E后进入B、BP、Ha、Hb之一的终局状态，等待新C出现后重启新周期。YG是从BS分离出的独立路径。

## 三、数据源规范
进行状态判定和评分计算时，**只能且必须**使用以下数据文件。禁止使用文件外的任何知识或记忆。

| 文件标识 | 关键字段 | 主要用途 |
|---------|---------|---------|
| `【大盘指数数据】` | 上证点位、成交额(沪+深)、上涨/下跌家数、20日均线 | 判定环境等级(S/A/B/C) |
| `【板块行情数据】` | 行业/概念板块涨幅、涨跌比、主力净流入 | 计算主线资金流入评分 |
| `【涨停个股数据】` | 涨停时间、连板数、封单额、所属板块 | 统计涨停数量；识别先锋/活口 |
| `【炸板个股数据】` | 炸板个股明细 | 排除炸板股 |
| `【跌停个股数据】` | 跌停个股明细 | 判定退潮确认(E) |
| `【核心中军行情数据】` | 中军涨跌幅、成交额 | 计算中军表现评分 |
| `【板块均线状态数据】` | 板块5/10/20日线状态 | 判定趋势支撑/破位 |
| `【产业催化电报】` | 政策/产业消息标题 | 计算政策强度评分 |
| `【强势股数据】` | 股票量比、趋势评分 | 识别趋势先锋(保留前50只) |
| `【历史策略快照】` | 上一交易日的全景表 | 获取锚点连续性与初始状态 |

## 四、复盘报告生成协议

### 4.0 初始状态获取
1. 从 `【历史策略快照】` 提取昨日全景表**最后一行**的“状态”、“主线锚点及阶段”、“候选池及阶段”作为今日初始状态。
2. **强制提取**主线锚点当前所处的阶段标记(如G2-3中的数字3)，作为今日n值计算的起点。
3. 若今日数据满足转移条件，更新状态；否则状态不变，天数(n)按交易日历递推增加1。

### 4.1 环境等级判定
1. 读取`【大盘指数数据】`的上证收盘点位、20日均线、成交额(沪+深)。
2. 读取`【涨停个股数据】`中的最高连板高度。
3. 按三项标准（指数>20日线，成交额>2万亿，连板高度>5）判定S/A/B/C。

### 4.2 主线评分流程
1. 从`【板块行情数据】`读取对应板块的涨幅、涨跌比、主力净流入。
2. 从`【涨停个股数据】`筛选该方向涨停股，统计数量。**归属以‘板块’字段为准**，一只股票最多归入一个方向。
3. 从`【核心中军行情数据】`筛选该方向中军，**取涨幅最大者**为评分依据，严禁取平均或最差。
4. 按五项权重计算得分。**评分过程不出现于正文。**

### 4.3 角色输出规范
每个主线方向，必须输出以下角色清单。**每类至少一个标的**。清单末尾注明“**当前最优标的：[标的名称]，理由：[基于风格优先级和数据的简短理由]**”。

| 角色 | 识别标准 |
|------|---------|
| **连板先锋** | 属该方向，连板数最高，封板时间早，封单强。优先10cm主板。 |
| **趋势先锋** | 属该方向，量比≥1.5，趋势评分最高，≤2板。优先100元以内。 |
| **核心中军** | 属该方向，市值最大且成交最活跃的趋势股。 |
| **弹性先锋** | 属该方向，最早涨停的20cm标的。 |
| **活口** | 属该方向，分歧日跌幅<3%，分时在均线上方。 |
| **卡位** | 龙头断板当日逆势涨停，次日往往无法延续→**不参与**。 |

- **趋势主导风格**下，优先趋势先锋 > 连板先锋 > 核心中军。
- **连板主导风格**下，优先连板先锋 > 趋势先锋 > 核心中军。

### 4.4 操作建议生成
1. 从全景表确定当前**状态标记**。
2. 从**2.1节状态速查表**检索对应**操作指令**。
3. 将操作指令与角色清单中的最优标的结合，填入全景表。

### 4.5 全景表输出格式
所有复盘报告必须包含此表，且作为下一日输入载体。

| 日期 | 指数量能 | 环境等级 | 状态 | 主线锚点及阶段 | 候选池及阶段 | 情绪与操作要点 |
|------|---------|---------|------|--------------|------------|--------------|
| (历史继承) | ... | ... | ... | ... | ... | ... |
| (今日) | 沪指涨跌幅%/全市场成交额(万亿) | 当前环境等级 | 当前状态标记 | 主线方向-状态标记 | 候选方向-状态标记 | 对应操作指令+最优标的 |
