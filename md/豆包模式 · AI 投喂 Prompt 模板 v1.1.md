# 豆包模式 · AI 投喂 Prompt 模板 v1.1

> **本文件是投喂给 DeepSeek 的指令说明书，也是我们后续所有自动化开发的蓝图依据。**

它定义了系统如何将数据、规则、状态拼接成 AI 能理解的文字，并规定 AI 必须输出的格式。后续所有自动化复盘和盘中监控，都将基于此文档组装 Prompt。

---

## 一、复盘投喂 Prompt

### 1.1 系统角色设定

Prompt 开头固定注入:

```
你是一个严格遵循豆包模式交易系统的首席策略分析师。
你精通题材周期、主线识别、双锚点机制和中军/先锋的定位。
你不依赖单一数据点，而是结合市场状态、产业背景和规则做出判断。
```

### 1.2 投喂数据清单

> 每次复盘由 Python 脚本 (`prompt_builder.py`) 将以下 **4 个模块** 拼成一段完整文本，发给 DeepSeek。

| 模块 | 数据来源 | 存放位置 | 负责人 |
|------|---------|---------|--------|
| **① 今日市场数据** | 指数、涨停、板块涨幅、中军Top20、强势股池等，由 `merge_replay.py` 生成 | `py/data/replay_full_YYYYMMDD.txt` | `prompt_builder.py` 读取后以纯文本注入 |
| **② 题材方向关键词** | 概念 → 关键词映射，用于 AI 辨识方向归属 | `py/config/config.json` → `cluster_keywords` | `prompt_builder.py` 读取后附在数据末尾 |
| **③ 规则手册** | 豆包模式全部量化规则 | `md/rules_full.md` | 直接附在 Prompt 中 |
| **④ 纵向市场快照** | 上个交易日的全景状态（锚点、候选池、阶段）| `py/config/session_snapshot.json` | `session_manager.py` 更新，`prompt_builder.py` 读取 |
| **⑤ 产业背景** | 当日财联社/21世纪等电报提炼的产业催化摘要 | `py/tools/market_context_builder.py` → `py/data/market_context_YYYYMMDD.txt` | 每天收盘后运行一次 |

### 1.3 输出格式要求

AI 必须严格按照以下结构输出 (可直接保存为最终策略文件):

```md
# 豆包模式 · 每日复盘清单 (YYYYMMDD)

## 1. 大盘状态
- 环境等级、量能、赚钱效应、20日线位置。

## 2. 情绪周期
- 涨停/炸板/跌停数据，连板高度，情绪定位。

## 3. 市场阶段与全景表
- 判断市场是主升、横盘还是主跌。
- **全景表格式** (必须展示主线演绎过程):
| 日期 | T+N | 指数 | 量能 | 主线/挑战者阶段 | 操作要点 |

## 4. 主线评估与候选池
- 当前核心主线的五项评分卡。
- 候选池内各方向的状态、阶段、评分、承载力。
- **角色定位**: 中军、趋势先锋、连板先锋。

## 5. 明日执行清单
- 根据买点规则 (C-A/B, D1, D2, D3) 生成具体标的和仓位建议。

## 6. 持仓处理与风险提示
- 对已有持仓的止损/止盈建议。
```

**AI 完成输出后，`session_manager.py` 将自动更新 `session_snapshot.json` 供下次复盘使用。**

## 二、盯盘投喂 Prompt

### 2.1 系统角色

```
你是一个豆包交易系统的实时监控员。你的任务是检查昨日策略中的买点条件是否满足。
```

### 2.2 投喂数据清单

| 模块 | 数据来源 | 负责人 |
|------|---------|--------|
| **① 盘中快照** | 上证/深证实时指数、涨幅前10概念板块、所有涨停个股、重点标的实时价/量 | `mid_day_monitor.py` |
| **② 昨日策略监控池** | 提取昨日 `strategy_YYYYMMDD.md` 中的“明日执行清单” | `mid_day_monitor.py` |
| **③ 分时数据 (可选)** | 标的的5分钟K线或分时图，东方财富 `secid=股票代码` 的K线接口 | 待开发 |

### 2.3 输出要求

AI 必须**简洁**回复，形式如下:

```
- 融捷股份: D2 买点已触发 (高开 3.5%, 竞价放量 42%, 现价 19.20)，建议仓位 1.5 成。
- 天齐锂业: D3 条件不满足 (未回踩 5 日线)。
- 北方稀土: D3 条件不满足 (放量下跌，观望)。
```

如果所有标的都未触发，回复: "当前无买点触发，继续观察。"

*`mid_day_monitor.py` 负责采集盘中快照、组装 Prompt、调用 API 并检查返回结果。若检测到买点触发，可调用 Windows 弹窗或语音播报。*
